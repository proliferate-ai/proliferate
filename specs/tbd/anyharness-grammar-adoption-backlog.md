# AnyHarness Grammar Adoption Backlog

Status: non-authoritative migration backlog. The operating law is
[../codebase/structures/anyharness/guides/mental-model.md](../codebase/structures/anyharness/guides/mental-model.md)
and the per-layer guides; this file is the work queue produced by a 12-agent
audit of the Rust codebase against that standard (2026-06-09, 144 findings:
34 high / 91 medium / 19 low). Line numbers will drift; re-verify before
fixing. Delete entries as they land; delete the file when it is empty.

## Recommended Order

- **Wave 0 (in flight):** agents domain regroup — the reference
  implementation of the standard.
- **Wave 1 — correctness patches (S/M, independent PRs):** the high-severity
  items marked CORRECTNESS below: env-map swap hazards, infra-errors-as-404,
  effects-before-validation in mobility archive install, permission denials
  miscategorized, control flow on error message text.
- **Wave 2 — the sessions/live spine:** SessionLaunch bundle + capability
  traits, latency-to-span migration, collapse the create relay.
- **Wave 3 — edge normalization:** one ApiError mechanism, From-impl
  migration, delete duplicate mappers, SessionView for the fetching mapper.
- **Wave 4 — god-use-case extractions:** workspace retire (3 copies to 1),
  fork, mobility preflight, subagent create.
- **Wave 5 — DTO twins (L):** runtime_config wire-types-as-rows, auth_config
  domain twins, mirror-struct collapses, per-domain wire().
