# Address GitHub PR comments

Use this skill when the task is to understand or address review feedback on a GitHub pull request.

1. Resolve the repository and pull request number.
2. Load unresolved review comments and any surrounding diff context available through the GitHub MCP server.
3. Group feedback by file, behavior, and reviewer intent. Distinguish requested changes from informational comments.
4. Before editing code, state the concrete comments you are addressing and any comments that need clarification.
5. After changes, summarize which review threads are handled and mention any remaining risks or tests that should run.
