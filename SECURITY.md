# Security Policy

Proliferate takes security reports seriously and appreciates responsible
disclosure.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

1. Preferred: Use GitHub Security Advisories ("Report a vulnerability") for this repository.
2. If you cannot use GitHub advisories, email [pablo@proliferate.com](mailto:pablo@proliferate.com).

When reporting, include:

- A description of the issue and potential impact
- Steps to reproduce (or a proof of concept)
- Affected versions / commit SHA if known
- Any mitigations or workarounds you found

## Scope

In scope:

- Vulnerabilities in this repository and its released artifacts

Out of scope:

- Social engineering and spam
- Denial-of-service reports
- Vulnerabilities in third-party services this project integrates with (Modal, E2B, GitHub, etc.)

## Coordinated Disclosure

We aim to acknowledge reports within 3 business days and will work with you on a
fix and a disclosure timeline.
