# Subagents Product MCP

Status: authoritative target definition for the Subagents product MCP,
subagent prompt behavior, and product/UI semantics.

## Purpose

The Subagents product MCP lets a parent agent delegate bounded same-workspace
work to child agent sessions. A subagent is not a separate runtime primitive.
It is a normal session with a durable parent-child link, product-scoped MCP
tools, provenance, wake scheduling, and UI presentation.

The design goal is simple for the parent agent:

```text
create a child
send it work
optionally wake me when it finishes
read the latest useful result
close it when I am done with it
```

The agent-facing API must hide transport and storage details. `sessionLinkId`
and `childSessionId` are internal/debug identifiers. The normal handle is
`subagentId`.

During the migration window, MCP responses may still include legacy
`sessionLinkId` and `childSessionId` fields so older transcript parsers, SDKs,
and in-flight agent sessions keep working. New callers should treat those
fields as compatibility aliases only. All examples and new code should prefer
`subagentId` plus `label`.

## Product Model

```text
Subagent
  subagentId
  label
  status
  harnessId
  appliedInitialConfig
  parentSessionId
  workspaceId
  child session reference
  wake state
  latest turn summary
  transcript/event cursors
  closed/deleted state
```

Identity fields:

| Field | Meaning | Normal exposure |
| --- | --- | --- |
| `subagentId` | Stable agent-facing and UI-facing handle for follow-up calls. | MCP args/responses, UI data model |
| `label` | Product title supplied by the creating agent or generated by the runtime. | Tool responses, transcript, tabs, sidebar, popovers |
| `avatarName` | Deterministic friendly display name, such as `Mary`. | UI hover/tooltip only |
| `color` | Deterministic visual color derived from `subagentId`. | UI only |
| `childSessionId` | Runtime session id for opening/routing the child session. | Internal/debug/details only |
| `sessionLinkId` | Durable parent-child relationship row id. | Internal/debug/details only |

`label` is the MCP field name, but product copy may call it the subagent title.
The label should be display-cased in UI when it was generated from a slug, for
example `api-surface-check` -> `API Surface Check`.

## MCP Identity

```text
id: subagents
owner: sessions/subagents
implementation: anyharness-lib/src/sessions/subagents/mcp/**
route slug: subagents
server name: proliferate-subagents
visibility: internal
default injection: parent sessions that may create same-workspace child sessions
```

The live session actor receives a final concrete MCP server config. It must not
decide whether subagents are enabled.

## Capability And Auth

Every MCP call is authorized against:

```text
workspace_id
parent_session_id
product_mcp_id: subagents
```

Child access is checked by the subagents domain on every child-targeting call.
A token minted for one parent session cannot operate on another parent's
subagents, even in the same workspace.

The shared product MCP token envelope belongs in:

```text
anyharness/crates/anyharness-lib/src/integrations/mcp/product_server/**
```

Subagent-specific scope construction and validation belongs in:

```text
anyharness/crates/anyharness-lib/src/sessions/subagents/mcp/auth.rs
```

## Tool Contract

All tools accept and return JSON objects. Tool results should be short enough
that the parent agent can understand the next action without inspecting raw
runtime state. Target-state child-targeting results include both `subagentId`
and `label`, so the parent agent always knows the stable handle and the human
name for the child it is managing.

Compatibility rule:

- accept `childSessionId` during migration when a caller does not yet know
  `subagentId`
- if both `subagentId` and `childSessionId` are supplied, they must resolve to
  the same linked child or the call returns a validation error
- return legacy ids only as compatibility fields; do not require new callers
  to store or echo them

Current migration response shape:

- create/send/status/wake/close responses include `subagentId` and `label`, and
  may also include `sessionLinkId` and `childSessionId`
- `read_subagent_latest_turns` returns `childTurnId`, `outcome`, `createdAt`,
  `childLastEventSeq`, `assistantText`, `toolErrors`, and `eventCount`
- `search_subagent_transcript` returns `query`, `seq`, `timestamp`, `turnId`,
  `itemId`, and `snippet`
- `read_subagent_events` is a debug escape hatch and may return only raw event
  cursors plus legacy child routing fields

These compatibility fields are not the agent-facing handle. The agent-facing
handle remains `subagentId`.

## Tool Call Presentation

MCP payloads stay machine-readable, but transcript/tool-call UI should render
subagent actions as product events instead of raw JSON whenever possible.

Create presentation:

```text
API Surface Check agent created
Running - Claude - Wake scheduled
Open agent session
```

Send presentation:

```text
Message sent to API Surface Check
Queued while the agent is running
Open agent session
```

Wake presentation:

```text
Wake scheduled for API Surface Check
```

Close presentation:

```text
API Surface Check agent deleted
```

The visible link should open the child agent session. UI should resolve the
open-session target from product state using `subagentId`; raw session/link ids
must stay hidden unless the user opens explicit debug/details UI.

### `get_subagent_launch_options`

Returns the launch catalog available to this parent session.

Args:

```json
{
  "harnessIds": ["claude"]
}
```

`harnessIds` is optional. If omitted, return the full available catalog for the
session. If present, return only matching harnesses and any unavailable reasons.

Returns:

```json
{
  "canCreate": true,
  "blockedReason": null,
  "limits": {
    "maxActiveSubagents": 8,
    "activeSubagents": 2,
    "grandchildrenAllowed": false
  },
  "defaultHarnessId": "claude",
  "harnesses": [
    {
      "harnessId": "claude",
      "displayName": "Claude",
      "defaultInitialConfig": {
        "model": "claude-sonnet-4-5",
        "mode": "default",
        "effort": "medium"
      },
      "initialConfigSchema": {
        "type": "object"
      },
      "summary": "Best for broad codebase exploration and implementation."
    }
  ]
}
```

There are no top-level `modelId` or `modeId` fields in subagent tools.
Harness-specific launch configuration is carried through `initialConfig`.

### `create_subagent`

Creates a durable child session, links it to the parent, starts the child, and
sends the initial prompt.

Args:

```json
{
  "label": "API Surface Check",
  "harnessId": "claude",
  "initialConfig": {
    "model": "claude-sonnet-4-5",
    "mode": "default",
    "effort": "medium"
  },
  "prompt": "Inspect the public API surface for mismatches...",
  "wakeOnCompletion": true
}
```

Required:

```text
prompt
```

Optional:

```text
label
harnessId
initialConfig
wakeOnCompletion
```

`prompt` is validated with `prompt.trim().is_empty()`, but the original prompt
string is preserved when it is sent to the child.

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "status": "running",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "appliedInitialConfig": {
    "model": "claude-sonnet-4-5",
    "mode": "default",
    "effort": "medium"
  },
  "wakeOnCompletion": {
    "scheduled": true
  },
  "next": {
    "readTool": "read_subagent_latest_turns",
    "subagentId": "subagent_abc123"
  }
}
```

Target-state normal responses do not need `childSessionId` or `sessionLinkId`.
During compatibility migration, responses may include them alongside
`subagentId`; consumers must use `subagentId` as the primary handle.

### `list_subagents`

Lists subagents owned by the parent session.

Args:

```json
{
  "includeClosed": false
}
```

Returns:

```json
{
  "subagents": [
    {
      "subagentId": "subagent_abc123",
      "label": "API Surface Check",
      "status": "running",
      "harnessId": "claude",
      "sessionLink": {
        "label": "Open agent session",
        "available": true
      },
      "wakeScheduled": true,
      "queuedMessages": 0,
      "latestTurn": null
    }
  ]
}
```

The list is ordered by active attention first, then running, then recently
updated, then closed when `includeClosed` is true.

### `send_subagent_message`

Sends a follow-up prompt to a subagent. If the child is currently running, the
message is queued. The tool should not fail solely because the child is busy.

Args:

```json
{
  "subagentId": "subagent_abc123",
  "prompt": "Also check the generated SDK types.",
  "wakeOnCompletion": true
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "status": "running",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "messageStatus": "queued",
  "wakeOnCompletion": {
    "scheduled": true
  }
}
```

### `schedule_subagent_wake`

Schedules a one-shot parent wake after the subagent's next newly completed
turn.

Args:

```json
{
  "subagentId": "subagent_abc123"
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "scheduled": true,
  "alreadyScheduled": false
}
```

Wake scheduling is explicit and not retroactive. `wakeOnCompletion` on
`create_subagent` and `send_subagent_message` is preferred when the wake should
be tied to the prompt being sent, because it avoids the race where the child
finishes before the parent schedules a separate wake.

### `get_subagent_status`

Returns the current live/product state for one subagent.

Args:

```json
{
  "subagentId": "subagent_abc123"
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "status": "running",
  "harnessId": "claude",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "appliedInitialConfig": {
    "model": "claude-sonnet-4-5",
    "mode": "default",
    "effort": "medium"
  },
  "wakeScheduled": true,
  "queuedMessages": 0,
  "latestTurn": null
}
```

This is a status read, not a transcript read. Use
`read_subagent_latest_turns` for results.

### `read_subagent_latest_turns`

Common result-reading path. Returns the last completed child turns in concise
form.

Args:

```json
{
  "subagentId": "subagent_abc123",
  "limit": 3
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "turns": [
    {
      "turnId": "turn_123",
      "status": "completed",
      "startedAt": "2026-05-13T18:00:00Z",
      "completedAt": "2026-05-13T18:03:42Z",
      "assistantSummary": "Found one SDK mismatch in create_subagent.",
      "importantToolErrors": []
    }
  ]
}
```

The result may include concise assistant output, outcome, timestamps, and
important tool errors. It should not stream raw low-level events.

During migration, this response may also include `sessionLinkId` and
`childSessionId` as legacy routing aliases. The stable fields remain
`subagentId` and `label`.

### `search_subagent_transcript`

Grep path for finding where a child mentioned something.

Args:

```json
{
  "subagentId": "subagent_abc123",
  "query": "wakeOnCompletion",
  "limit": 10
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "matches": [
    {
      "turnId": "turn_123",
      "eventSeq": 42,
      "speaker": "assistant",
      "snippet": "wakeOnCompletion should be exposed on create and send..."
    }
  ]
}
```

During migration, this response may also include `sessionLinkId`,
`childSessionId`, and the echoed `query`.

### `read_subagent_events`

Advanced/debug escape hatch. Returns bounded sanitized child events.

Args:

```json
{
  "subagentId": "subagent_abc123",
  "sinceSeq": 42,
  "limit": 100
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "events": [],
  "nextCursor": {
    "sinceSeq": 87
  }
}
```

Agents should prefer `read_subagent_latest_turns` unless they need event-level
debugging.

### `close_subagent`

Removes a subagent from active delegated-work context. If the subagent is
running or has queued work, the runtime ends that active work as part of the
close operation.

Args:

```json
{
  "subagentId": "subagent_abc123"
}
```

Returns:

```json
{
  "subagentId": "subagent_abc123",
  "label": "API Surface Check",
  "sessionLink": {
    "label": "Open agent session",
    "available": true
  },
  "closed": true,
  "activeWorkEnded": true
}
```

Closing is not transcript deletion. The child session and historical artifacts
remain available through history/debug surfaces according to retention policy.

Close ordering is intentionally retryable: the runtime closes the child session
graph first, including any delegated descendants and product close hooks, then
marks the parent-child link closed. If closing the live session fails, the
active link remains discoverable so a later close call can retry rather than
orphaning hidden work.

## Prompt And Skill Contract

The parent session must not receive a baked-in system prompt explaining
subagent tools. Workflow guidance belongs in a product skill.

Required skill:

```text
skill id: proliferate.subagents.workflow
display name: Subagent workflow
description: Use Proliferate subagent MCP tools for bounded parallel work,
  result reads, wake scheduling, and cleanup.
```

The skill should teach:

- why Proliferate `subagents` MCP tools are preferred for same-workspace
  delegation when provider-native or internal subagent tools overlap
- when to delegate
- how to choose labels
- when to call `get_subagent_launch_options`
- why `wakeOnCompletion` is preferred for prompt-tied wakes
- how to read results with `read_subagent_latest_turns`
- when to use transcript search or raw events
- how to close stale subagents
- how to avoid duplicating work already delegated

Parent-to-child prompt rules:

- Preserve the exact prompt text supplied to `create_subagent` or
  `send_subagent_message` after non-empty validation.
- Do not wrap the prompt with hidden instructions.
- Record provenance as agent-originated prompt metadata.
- UI may present "sent by parent agent" using provenance, but the child should
  receive the actual prompt as the author supplied it.

Wake prompt text:

```text
Subagent "API Surface Check" finished a turn. Outcome: completed.

Use read_subagent_latest_turns with subagentId "subagent_abc123" before continuing.
```

Wake prompts should include:

- subagent label
- stable `subagentId`
- turn outcome as runtime status, not task success
- the recommended next read tool

Wake prompts should not include:

- `childSessionId`
- `sessionLinkId`
- raw event cursor unless the wake is explicitly a debug wake
- long child transcript excerpts

## UI Contract

The UI should present subagents as delegated work, not as MCP plumbing.

Surfaces:

```text
Tab strip
  lightweight delegated-work presence on the right side of the parent tab

Composer Agents popover
  active delegated-work inbox for the current session

Sidebar/session hierarchy
  durable navigation map of parent and child sessions

Transcript
  durable receipts for creation, completion, wake, and result artifacts

Details surface
  focused inspect/manage view for one subagent
```

Display rules:

- Tabs, sidebar, popovers, and transcript use `label`/title.
- Friendly avatar names such as `Mary` are hover-only texture.
- Deterministic color/avatar assignment is client presentation and is never
  required for MCP calls.
- The tab close control lives on the left side of the tab, leaving the right
  side available for delegated-work indicators.
- The right side of a parent tab may show up to three overlapping delegated
  agent bubbles plus a count overflow.
- The parent tab is the anchor; child-agent bubbles and any open child-agent
  tabs attach to the right of the parent, but the parent itself is not a member
  of the child-agent group.
- Opening a subagent session inserts or moves its tab into the contiguous
  child-agent run immediately to the right of the parent tab.
- Clicking the cluster opens the same delegated-work model used by the
  composer Agents popover.

Deletion rules:

- Closing a visible tab is distinct from deleting a delegated agent.
- Deleting a subagent from a delegated-work popover deletes/closes that
  delegated agent relationship.
- If the deleted subagent is active, confirm that active work will end.
- Deleting a completed subagent may be immediate.
- Closing/deleting a parent session with active child sessions must confirm
  that active child work will be ended.

## Cowork Compatibility

Cowork sessions should reuse the same delegated child-session lifecycle:

```text
link
prompt provenance
wake scheduling
completion tracking
latest-turn reads
close/delete semantics
UI identity
```

Cowork adds workspace selection/creation. It should not invent a second
subagent lifecycle. If a future cowork-facing MCP exposes the same child-agent
calls across workspaces, the extra argument should be a workspace scope/handle,
not a second set of lifecycle concepts. The target cowork cleanup is defined in
[cowork.md](cowork.md).

## Acceptance

Done when:

- parent agents use `subagentId` for all common follow-up calls
- common tool responses include `subagentId` and `label`; raw child/session
  link ids are compatibility/debug fields only
- launch config is carried through `initialConfig`
- `wakeOnCompletion` is advertised and works on create/send
- `read_subagent_latest_turns` is the normal result path
- transcript search and raw event reads are separate tools
- closing a subagent removes it from active delegated-work state
- close cascades through delegated descendants and only marks links closed
  after session close succeeds
- subagent workflow guidance is delivered as a skill, not parent system text
- parent-to-child prompt text is preserved after non-empty validation
- UI uses title/label as the serious name and avatar names as hover-only
  texture
