# Mockups

Use `settings-sample.html` as visual and interaction context for the
Settings/Admin IA work.

It is not an implementation source file. Treat it as a reference artifact for:

- settings sidebar structure;
- Agents, Agent Defaults, and Agent Authentication layout;
- Environments, Shared Environments, Compute, and Slack Bot page direction;
- card, badge, tag, list/detail, and add-row conventions.

When writing or cleaning the specs, preserve the product intent from this
mockup, but still check actual frontend area docs and current `apps/desktop/src/**`
patterns before naming concrete implementation files.
